<p>New maintenance releases for the Redmine 6.1, 6.0, and 5.1 series are now available to <a class="wiki-page" href="https://www.redmine.org/projects/redmine/wiki/Download">Download</a>. These releases address multiple security vulnerabilities along with various bug fixes and improvements.</p> <strong>Security Fixes</strong>:<br />All three versions (<a class="version" href="https://www.redmine.org/versions/224">6.1.3</a>, <a class="version" href="https://www.redmine.org/versions/223">6.0.10</a>, and <a class="version" href="https://www.redmine.org/versions/225">5.1.13</a>) include the following security fixes: <ul> <li>Defect #43951: Bulk attachment download bypasses View files permission for project/version attachments</li> <li>Defect #44109: PreAuth leak name of private Projects</li> <li>Defect #44118: Any project member with <code>add_issue_notes</code> permission can add notes to private issues they cannot view, via the MailHandler reply dispatch</li> <li>Defect #44138: Stored XSS in Textile formatter due to <code>restore_redmine_links</code></li> <li>Defect #44145: PostScript execution in <code>Redmine::Thumbnail.generate</code> via <code>%% DSC-comment</code> prefix</li> <li>Defect #44146: Time-entry API hidden custom-field leak</li> </ul> Versions <a class="version" href="https://www.redmine.org/versions/224">6.1.3</a> and <a class="version" href="https://www.redmine.org/versions/223">6.0.10</a> also include: <ul> <li>Patch <a class="issue tracker-3 status-5 priority-4 priority-default closed" title="Patch: Improve the `config.filter_parameters` setting (Closed)" href="https://www.redmine.org/issues/43986">#43986</a>: Improve the <code>config.filter_parameters</code> setting</li> </ul> Version <a class="version" href="https://www.redmine.org/versions/224">6.1.3</a> also includes: <ul> <li>Defect #44174: OAuth scope enforcement bypass in user account</li> </ul> <p>You can find the new versions in the <a class="wiki-page" href="https://www.redmine.org/projects/redmine/wiki/Download">Download</a> section. For a complete list of changes, please review the detailed <a class="wiki-page" href="https://www.redmine.org/projects/redmine/wiki/Changelog">Changelog</a> for each version.</p> <p>Many thanks to all the contributors who helped with these releases, especially those who responsibly reported the vulnerabilities and to <a class="user active" href="https://www.redmine.org/users/2784">Holger Just</a>, <a class="user active" href="https://www.redmine.org/users/40856">Jens Krämer</a>, and to <a class="user active" href="https://www.redmine.org/users/332">Go MAEDA</a> for their continuous work on these security issues.</p>
