<p>Maintenance releases <a class="version" href="https://www.redmine.org/versions/213">6.0.7</a>, <a class="version" href="https://www.redmine.org/versions/214">5.1.10</a> and <a class="version" href="https://www.redmine.org/versions/215">5.0.14</a> are now available to <a class="wiki-page" href="https://www.redmine.org/projects/redmine/wiki/Download">Download</a>, bringing a total of 16 bug fixes (<a class="wiki-page" href="https://www.redmine.org/projects/redmine/wiki/Changelog">Changelog</a>).</p> <p><strong>Security fixes:</strong></p> All versions contain the following security fixes: <ul> <li>Defect #42998: Username and password stored in login form</li> <li>Defect #43083: Information disclosure in Two-Factor Authentication</li> <li>Defect #43161: When copying issues, all existing custom values are set to the new issue without sufficient validation</li> </ul> <p>Starting with these versions, a new security measure has been implemented in #42998 to improve how Redmine handles sensitive information. The <code>no-store</code> cache header has been added to following forms: login, lost password, change password, sudo pages, auth_source, user, repository and accounts#register.</p> <p>Thanks everyone for their contributions.</p> <p><strong>A Note on the End of Life for Redmine 5.0</strong></p> <p>With the upcoming release of Redmine <a class="version" href="https://www.redmine.org/versions/198">6.1.0</a> later today, we want to remind everyone that this will mark the end of life for the Redmine 5.0 series. If you are currently using a version in the 5.0 branch, we highly recommend you plan to upgrade soon to continue receiving updates and security patches.</p>
